# IPAM Autopilot Provider

The IPAM Autopilot provider allocates and manages IP CIDR ranges via a running [IPAM Autopilot](https://github.com/boozt-platform/ipam-autopilot) backend.

Use the provider to dynamically allocate non-overlapping subnets for GCP VPCs — GKE node pools, pod ranges, services, and more — without hardcoding CIDRs across environments.

## Authentication

The provider authenticates against the Cloud Run backend using [Application Default Credentials](https://cloud.google.com/docs/authentication/application-default-credentials).

**Local development:**

```bash
gcloud auth application-default login
```

**CI/CD — set `IPAM_IDENTITY_TOKEN` to a valid Google identity token:**

```bash
export IPAM_IDENTITY_TOKEN=$(gcloud auth print-identity-token \
  --audiences="https://your-ipam-cloud-run-url")
```

## Example Usage

```terraform
provider "ipam" {
  url = "https://your-ipam-cloud-run-url"
  # Alternatively, set the IPAM_URL environment variable.
}
```
